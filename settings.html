Roles & Career Levels
WorkDesk uses a two-axis model: Roles define what a user can do across the platform, while Career Levels indicate seniority within an employee's track.
🏷️ Organizational Roles
Role 1
Entry Level
Front-line staff. Can view and submit requests; modifications require supervisor approval.
Role 2
Supervisor
Approves requests from Entry Level staff. Limited edit access with manager oversight.
Role 3
Manager
Full team management. Can approve supervisor-escalated requests. Requires Senior Manager approval for own modifications.
Role 4
Senior Manager
Cross-team oversight. Approves Manager-level requests. Requires General Manager sign-off for system-level changes.
Role 5
General Manager
Organization-wide authority. Final approver within the org. Can delegate Admin functions.
Admin
Organization Admin
Manages all org settings, user access, and configurations within their organization.
★ Super
Super Admin
Platform-level access. Can create/modify/remove organizations and assign Org Admins. CEO-level platform control.
📊 Career Levels (1–8)
Career levels measure seniority and growth within a role track. They determine access scope and base permissions but are always coupled with the employee's Role for final access decisions.

Access & Permissions
Permissions are derived from each employee's Role. All modifications flow through an approval chain based on the requester's role. Changes are logged as immutable footprints visible only to higher-access users.
ℹ️ Approval Chain
Entry Level → Supervisor → Manager → Senior Manager → General Manager → Admin → Super Admin. Each level can approve requests from the level directly below them. Managers and above require approval from their own next level for self-modifications.
⚠️ Attendance Records
AUX/attendance records are immutable once recorded. Entry Level & above may submit a Modification Request with a written explanation and supporting attachment. Requests require approval from the next-level manager before any correction is logged.

Organization Settings
Manage your organization's profile and configuration. Only Organization Admins and Super Admins can modify these settings.
🔒 Admin Access Required
Changes to organization settings are logged and visible to all Admin-level and above users.
Admin Users
Organization Admins
Admins have full access to all organization settings and can manage user roles. Only Super Admins can add or remove Admin access.
