Database & Backup Setup
Follow these steps to connect Cloudflare D1, apply the schema, and seed initial data.
Checking…
  1. 1
    Create the D1 database
    Run the following Wrangler command from the project root to provision your Cloudflare D1 database.
    wrangler d1 create workdesk-db
    Copy the database_id value that is printed.
  2. 2
    Update wrangler.toml
    Uncomment the [[d1_databases]] block in wrangler.toml and paste your database_id.
    [[d1_databases]] binding = "DB" database_name = "workdesk-db" database_id = "<YOUR_D1_DATABASE_ID>"
  3. 3
    Apply the database schema
    Run the included schema.sql to create all tables.
    wrangler d1 execute workdesk-db --file=./schema.sql
  4. 4
    Seed initial / backup data
    Load the included seed.sql to pre-populate demo or initial data.
    wrangler d1 execute workdesk-db --file=./seed.sql
  5. 5
    Create KV namespace for sessions
    Creates the KV store used for authenticated session tokens.
    wrangler kv:namespace create "SESSIONS"
    Then uncomment and update the [[kv_namespaces]] block in wrangler.toml.
  6. 6
    Set the token-signing secret
    Replace the default demo secret with a strong random value.
    wrangler secret put TOKEN_SECRET
  7. 7
    Deploy to Cloudflare Pages
    Push the final build to your Cloudflare Pages project.
    wrangler pages deploy .
